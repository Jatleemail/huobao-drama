# 角色与场景提取指南

## 角色提取规范

提取的角色信息包含：
- **姓名**：角色全名
- **角色定位**：主角/配角/龙套
- **外貌描写**：性别、年龄、体型、面部特征、发型、着装（300-500字）
  - 体型：身高、体型（瘦/匀称/魁梧/肥胖）、体态特征
  - 面部：脸型、五官特征、肤色、年龄痕迹
  - 发型：长度、颜色、质感、造型
  - 服装：风格、颜色、材质、标志性配饰
  - 标志性特征：疤痕、纹身、独特饰品等（可选）
- **性格特点**：核心性格标签（3-5个关键词）
- **角色描述**：背景故事和关系

### 外貌描写示例（合格标准）

> 青年男性，约 20 岁，身形清瘦，瓜子脸，剑眉星目，皮肤白皙。黑色长发以布带束于脑后。身着灰色僧袍，脚穿草鞋。左手腕有一串檀木佛珠。

## 场景提取规范

提取的场景/背景信息包含：
- **地点**：具体场所名称
- **时间**：时间段和光线条件
- **光线描写**：
  - 自然光：时间段 + 光源方向 + 色温（如：黄昏侧光，暖金色）
  - 室内光：光源类型 + 强度 + 阴影特征（如：烛光摇曳，暖黄色，墙面长影）
  - 特殊天气：雨天/雪天/雾天 → 对应散射光和能见度描述
- **氛围**：环境氛围描述
- **提示词**：用于AI图片生成的英文提示词（纯背景，不含人物）

## 道具提取规范

提取的道具信息包含：
- **名称**：道具名（简洁准确，2-6字）
- **类型**：武器 / 文件书信 / 食物饮品 / 交通工具 / 装饰品 / 科技设备 / 自然物品 / 其他
- **描述**：外观和用途（≥50字，含材质、颜色、形状、大小、标志性细节）
- **图片提示词**：英文，纯物品产品图风格，clean product shot, no people, no hands, isolated on neutral background

### 道具提取触发条件

满足以下**任一**条件即提取：
1. 被角色持有/使用且有特写描述
2. 推动剧情发展的关键物品（如信物、武器、钥匙、书信）
3. 反复出现（≥2 次）的环境物件或标志性物品

### 道具图片提示词规范

格式：
```
A clean product-shot of [道具名], [材质+颜色+形状], [关键细节],
isolated on neutral background, studio lighting, high quality, detailed,
no people, no hands, no text, no watermark.
```

示例：
> A clean product-shot of an ancient bronze sword, bronze blade with green patina, dragon-head hilt carved from jade, blade length approx 80cm, double-edged with subtle blood groove, isolated on dark gradient background, dramatic rim lighting highlighting edge, high quality, detailed, no people, no hands, no text, no watermark.

### 道具去重规则
- 按**名称**精确匹配
- 同名道具 → 合并信息，取更详细描述和提示词
- 新类型覆盖旧类型（如从「其他」升级为「武器」）
- 未匹配 → 正常新增

### 工作流程中的道具处理
角色和场景提取完成后，继续处理道具：
1. 调用 `read_existing_props` 读取项目已有道具
2. 分析本集涉及的道具，逐个判断新增或去重
3. 调用 `save_dedup_props` 保存道具（自动去重合并）

## 去重与合并细则

### 角色去重
- 按**姓名**精确匹配（同名不同姓视为不同角色）
- 匹配到同名角色时**合并信息**：
  - 外貌/性格较新提取更详细 → 更新对应字段
  - 新旧信息冲突 → 优先当前集描述，在角色描述中标注差异（如”[本集更新]...”）
  - 角色定位升级（龙套→配角）→ 取更高级别
- 未匹配到 → 正常新增并关联当前集

### 场景去重
- 按「地点名 + 时间段」精确匹配
- 同地点+同时段 → 复用已有场景，不重复创建；若新描述更丰富则更新
- 同地点+不同时段 → 视为不同场景（光线/氛围差异大）
- **复用场景也必须关联到当前集**（否则该集看不到已有场景）

## 提取优先级

- **角色**：有台词 > 有重要动作 > 被其他角色明确提及且影响剧情 > 龙套
- **场景**：主要事件发生地 > 过渡场景 > 仅被提及但未实际出现
- 只提取当前集真实出现或被明确提及的内容，不虚构或推测

## 关联规则

- 角色和场景提取后**必须**通过 save 工具关联到当前集
- 复用已有角色/场景时也需要建立当前集关联
- 关联后该集才能在使用角色/场景列表时看到这些数据
